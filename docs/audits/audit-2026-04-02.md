## Tech Stack

FastAPI 0.109 + Uvicorn + SQLite (WAL mode) + Jinja2 templates. Deployed to Fly.io.
Python 3.12, 19 dependencies (all current except slowapi 0.1.9 — stale but stable).

## Architecture

**Strengths:**
- Clean module separation: scoring.py (medals), sync.py (QRZ/LoTW), notifications.py (push/email/Discord), auth.py (sessions/lockout)
- Thread-safe SQLite with WAL mode, exclusive transactions for medal recomputation
- Background sync via subprocess (non-blocking), async email delivery
- 86+ configuration options via environment variables with sensible defaults
- Distribution support: `olympics` and `coolcontest` theme variants

**Weaknesses:**
- **main.py (8,760 lines, 62 endpoints)** — God file. Should split into route modules (public, auth, user, admin, api).
- **Embedded database migrations** — 11+ migrations in init_db() (~700 lines). No rollback capability. Should use Alembic.
- **Email logic scattered** across main.py, notifications.py, and email_service.py — unclear separation.
- **Sync subprocess** approach spawns a new process per sync cycle. No cleanup timeout guarantee.

## Security

**Grade: A-** — Production-ready security posture.

| Area | Implementation | Status |
|------|---------------|--------|
| Password storage | bcrypt (12 rounds, configurable) | Secure |
| API credential encryption | PBKDF2 (100K iterations) + Fernet, per-deployment salt | Secure |
| Session management | secrets.token_urlsafe(32), httponly cookies, 30-day expiry | Secure |
| Account lockout | 5 failed attempts -> 15-min lockout | Secure |
| CSRF protection | Token in cookie + form validation middleware | Secure |
| SQL injection | Parameterized queries throughout (~90%+ of queries) | Secure |
| XSS prevention | Jinja2 autoescape + manual escaping in filters | Secure |
| Security headers | CSP, HSTS, X-Frame-Options, X-Content-Type-Options, Referrer-Policy | Excellent |
| Sensitive data logging | Redaction filter for passwords/API keys in httpx logs | Secure |
| Rate limiting | slowapi on signup (5/min), login (10/min) | Good |

**Minor gaps:**
- No 2FA/MFA (acceptable for ham radio community app)
- Password reset endpoint not rate-limited (email spam risk)
- Callsign input not regex-validated (relies on `.upper().strip()` only)

## Dependencies

All 19 dependencies current. No known CVEs. Only concern: slowapi (0.1.9) last updated 2021 — stable but unmaintained.

## CI/CD

- GitHub Actions: pytest on push/PR to main, auto-deploy to Fly.io on merge
- Python 3.12 with pip caching
- **Gaps:** No coverage reporting, no linting (black/flake8/mypy), no security scanning (bandit)

## Database

SQLite with WAL mode, foreign keys, CHECK constraints, proper indexes. 11+ schema migrations embedded in init_db().

**Key tables:** olympiads, sports, matches, competitors, qsos, medals, records, teams, team_members, push_subscriptions, audit_log, resource_files.

**Concerns:** No rollback capability for migrations. No version tracking table. init_db() is ~700 lines. Should migrate to Alembic for safer deployments.
